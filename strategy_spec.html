▾

概要

双方向グリッド（Long/Short 同時稼働）。ATR幅でグリッドを形成し、Choppiness Index（レンジ相場フィルター）が閾値以上の時のみエントリー。各ポジションに TP のみ設定（SL なし）。最大段数（max_levels）到達後 48時間経過で強制決済（B48 Exit）。方向別の含み損が FLOAT_STOP を下回った場合は即時強制決済（Float Stop）。

v6（2026-06-02 パラメータ最適化）：float-stop込み2年BTで CHFJPY/NZDJPY が負エッジと判明。真因は max_levels=7 で含み損が深く積まれ float-stop が先発動し B48がデッドだったこと。max_levels 浅化 + ci_threshold 厳格化で B48 を主決済化。

v7（2026-06-02 float_stop結合最適化）：float_stopは深いラダーのテール保険としてのみ有効。浅いラダーでは緩め固定が正解。NZDJPY -500k→-1.0M で lv7 復活、AUDCAD lv5/fs-750k に改善。

v8（2026-06-13 改善ツールキット導入・Dukascopy 11年BT）：PF期待値バー（IS-selectable ∧ OOS>1.2 ∧ wfoMin>1.0）を超えた全ペアを推奨設定でフォワードテスト可能に。新ノブ（既定OFF=v7挙動）＝mom_thr（24h ATR正規化リターンの方向ゲート＝ナイフ掴み追加を遮断）/ mom120_thr（5日地平線・EURGBP）/ cull_frac（含み損が予算超で最悪レッグ1本のみ段階決済）/ taper（深レベルのロット縮小）/ dir_mode（both / long_only=carryグリッド USDJPY・NZDJPY / regime_short=上昇局面のみ新規short停止 AUDCAD・AUDNZD）/ short_lot_mult・tp_mult（EURGBP）。救済2類型＝①JPYクロスcarry-grid(long-only) ②相関クロスgrid(両側/レジーム+combo)。GBPJPY/CHFJPY/EURUSDは構造的に救済不可（No-Go据置・非起動）。

エントリーフロー（60秒ループ、毎サイクル実行）

  1. フィルター判定
    CI(D1,14) を計算 → CI > CI_th（ペア別） を満たさない場合は filter_block ci=XX.X ログを出力してエントリーをスキップ
  2. 初回エントリー
    フィルター通過 ＋ その方向のポジション数 = 0 → 成行注文（Long は Ask、Short は Bid）。
    TP = エントリー価格 ± grid_width を同時設定。SL = 設定なし
  3. グリッド追加エントリー
    フィルター通過 ＋ ポジション数 ≥ 1 かつ < max_levels の場合：
     Long: 現在値（Mid） ≤ 最安エントリー価格 − grid_width → 成行 Buy 追加（平均取得単価の分散）
     Short: 現在値（Mid） ≥ 最高エントリー価格 + grid_width → 成行 Sell 追加
  4. 最大段数到達
    ポジション数 ≥ max_levels → 追加エントリーは行わず B48 タイマーを起動

決済フロー（優先順位順）

  1. TP（個別・最優先）
    各ポジションの TP 価格 = エントリー ± grid_width に到達 → MT5 が自動執行（個別）。1段分の利益を確定。
    TP 発火でカウントが max_levels を下回ると B48 タイマーをリセット（回復チャンスを確保）
  2. Float Stop（方向別・緊急決済）
    毎サイクル最初にチェック。方向別の含み損合計 < FLOAT_STOP_JPY → その方向の全ポジションを即時成行決済 ＋ B48 タイマーリセット。
    設計意図：深いラダー（lv7）向けテール保険。浅いラダー（CHFJPY lv3）では -1.5M 設定で非発動が最良設計（きつくすると回復可能ポジを切り PF・DD 悪化）
  3. B48 Exit（方向別・時間決済）
    max_levels 到達時刻から 48 時間経過 → その方向の全ポジションを成行決済。
    設計意図：損切りではなく「時間制限のある保有」。ラダーを浅く（lv3/5）することで Float Stop より先に B48 が発動し、単発損が緩やかに収まる（v6の核心）
  4. DD_DAY / DD_WEEK サーキットブレーカー（全方向）
    当日の確定損益 < DD_DAY、または当週の確定損益 < DD_WEEK → Long/Short 両方の全ポジションを成行決済 ＋ 以降のエントリーを当日/当週ブロック。
    注意：v7 で NZDJPY/AUDCAD の DD_DAY/DD_WEEK 値を緩和済み（新 float_stop の正常決済が完了する前にブレーカーが先行しないよう調整）
