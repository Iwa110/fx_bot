# AUDCAD(4h) 平均回帰・3段不等分割 — 実運用計画

作成 2026-06-30。検証フェーズ(Phase1-7 + ストレステスト)完了を受けた実稼働計画。
全数値は `optimizer/dynamic_lot_mr_bt.py` / `audcad_stress_test.py` の BT 由来(IS=2015-2021 /
OOS=2022-2026, フルコスト控除, Lookahead排除, 次足始値約定)。

---

## 1. 確定戦略仕様(凍結)

| 項目 | 値 |
|---|---|
| 銘柄 | **AUDCAD** (相関クロス=AUD/CADが資源・USDドライバ共有→独立トレンド乏しく構造的レンジ) |
| 時間足 | **4h** (1h比でノイズ減・フォールド頑健性向上: wfoMin 0.92→1.81) |
| シグナル | Z-score = (close − SMA40) / SD40。確定足で判定、次足始値で約定 |
| エントリー | **3段・不等分割(構成A)**: Tier1 \|Z\|≥2.0=0.2lot / Tier2 \|Z\|≥2.5=0.3lot / Tier3 \|Z\|≥3.0=0.5lot (最大合計=1.0) |
| 利確(TP) | 全段を **SMA(Z=0)回帰で一括決済**(構成A。4hでは部分利確Bより一括Aが優位) |
| 損切り(SL) | タイムストップ **48本(=8日)** or ハードストップ **\|Z\|≥4.5** で全決済 |
| 高ボラ・スロットル | エントリー時 **ATRパーセンタイル≥0.7 なら全段lotを×0.5**(エントリーは止めない=エッジ維持) |
| 方向 | long/short 両側(レンジ回帰) |

**根拠**: 検証で「フィルタ(エントリー選別)はPF不変〜悪化、配分(深Z加重・高ボラ減量)はPareto改善」が
一貫。本仕様は配分のみでエッジを最大化した形。`vol_throttle` 採用は ATR≥0.7→0.5x が
maxDD −42% / MC95 −37% / full PF 1.56→1.61 / OOS 2.59→2.57 の clean win であることを実測確認済。

## 2. 期待値の前提(実運用基準＝ボトムPFで設計)

> **ヘッドラインの OOS PF 2.60 を基準にしない。全期間 PF と MC 最悪DD を基準にする。**

| 指標 | throttle後の値 | 設計上の扱い |
|---|---|---|
| 全期間PF(2015-2026) | **1.61** | これを運用基準とする(OOSの2.57はボーナス=上振れ) |
| OOS PF(2022-2026) | 2.57 | 直近4年の順風レジーム。再現を前提にしない |
| IS PF(2015-2021) | ~1.2 | マクロ逆風期の現実。**純損失年=2015/2017/2020** |
| 勝率 | ~71% | 小さい勝ち多数 |
| 歪度 | **−2.5(損失側テール)** | 平均回帰の構造。稀に大負け(worst単発 −280 lot-pip)を受容 |
| MC maxDD 95%ile | **398 lot-pip** | サイジングの基準(下記) |
| MC maxDD 99%ile | ~470 lot-pip | 資本余力の下限チェック |

**マクロ高ボラ年(2015/2017/2020)の年次純損失は戦略の構造的コスト(保険料)**であり、
バグでも劣化でもない。平均回帰は「ボラ収束期に積み上げ、構造変化期に耐える」もの。

## 3. サイジング(MC 95%ile DD 基準)

AUDCAD pip価値: 1.0lot(10万通貨)で 1pip = 10 CAD ≈ **1,080円**(CADJPY≈108)。
→ throttle後 **MC95 DD = 398 lot-pip ≈ 430,000円 / 最大1.0lot枠**。

**安全lotスケール = (許容最大DD額) ÷ 430,000円**

| 自己資本 | 許容DD(資本の20%) | lotスケール | 実段別lot(T1/T2/T3) |
|---|---|---|---|
| 100万円 | 20万円 | 0.46 | 0.09 / 0.14 / 0.23 |
| 300万円 | 60万円 | 1.39 | 0.28 / 0.42 / 0.70 |
| 500万円 | 100万円 | 2.32 | 0.46 / 0.70 / 1.16 |

- **初期は上記の半分(=許容DD10%基準)で開始**し、フォワード一致を確認後に漸増。
- 99%ile(~470 lot-pip≒508k円)が許容資本を超えない範囲に必ず収める。
- レバレッジ上限・証拠金は別途確認(最大合計1.0lot×スケール分の必要証拠金)。

## 4. キルスイッチ(運用停止基準)— 事前確定

以下のいずれかで **自動停止 → 前提条件(相関・平均回帰構造)の再検証**:

1. **ローリング12ヶ月の実現PF < 1.0**(=直近1年で平均回帰エッジが消失)。
2. **現在ドローダウンが MC 95%ile DD(throttle後 398 lot-pip ≒ 43万円/lot×スケール)を超過**。
3. **構造前提の崩壊**: AUD/CAD の経済的同一ドライバ関係が壊れる兆候
   (例: 一方の中銀が極端な政策乖離 → クロスが構造トレンド化)。月次で AUDCAD の
   年次 trend_atr / path_eff を監視し、Goエンベロープ(年純変位ATR比が低い)を外れたら警告。

停止後は「再最適化」でなく「前提が生きているかの再検証」。生きていれば再開、崩壊なら撤退。

## 6. 実装(VPS / MT5)

- 新規 `vps/mr_monitor.py`。**magic=20260050 / STRATEGY_TAG='MR_AC'**(grid系20260030-038と分離)。
- **実行タイミング = 4h足確定時**(00/04/08/12/16/20 UTC)。Task Scheduler で 4h境界の数分後に起動、
  または毎時起動し「未処理の確定4h足」だけ評価(重複エントリー防止)。
- 状態管理: 建玉(段別エントリー価格・lot)、平均建値、タイムストップ起点バー、
  Z-score/SMA40/SD40/ATRパーセンタイルを毎回再計算(t-1確定値)。
- ブローカー: demo=Axiory/Exness。MT5 `copy_rates` で 4h(or 1hを4h集約)を取得。
- ログ: heartbeat 4h毎(`pos=段数/価格/Z`)、約定時に entry_t/Z/lot/throttle適用有無を記録し
  BT照合に使う。

### スリッページ管理(重要)
- AUDCAD は相対的に**スプレッド広め**(BT想定往復2.0pip)。4h足の始値約定なので**bar境界の
  ギャップ・低流動性時間帯(東京早朝など)を避ける**: エントリーは bar 確定直後の流動性がある
  窓に限定するか、成行に最大スリッページ上限(例 1.5pip)を設定。
- ハードストップ(\|Z\|≥4.5)は急変時にスリッページが乗る → BT は閾値到達価格で約定想定。
  実機の単発損が BT 比 1.3倍を超える頻度を監視(grid の float_stop 教訓: テールは閾値で
  完全には止まらない)。
- スリッページが想定の 1.3倍を恒常的に超える場合はブローカー/執行時間帯を見直し(キルスイッチ②と別管理)。

## 7. 留保・既知のリスク

- **OOS の順風(2022-26)が続く保証はない**。設計は全期間PF1.61・MC95DDで保守的に。
- **2015級の突発政策ジャンプ**(CHF unpeg型)は事前検出不能。\|Z\|≥4.5 ハードストップでも
  ギャップ貫通しうる → 最大1.0lot枠を厳守し、過大lot投入を禁止。
- **単一銘柄集中**。分散したい場合は別途検証済みの相関クロス(EURGBPは1hでIS<1.0で不採用、
  AUDNZDは4hでPF1.13と限界的)。現状 AUDCAD(4h)が単独で最頑健。
- 本計画は BT 由来。**①demo フォワード完了が実マネーの絶対条件**。
