## 0. レバレッジ・複利の正しい理解（最重要・誤解厳禁）

- **拘束条件は「証拠金」でなく「最大DDに耐える必要資本 req_cap_99」**。高レバで証拠金が
  足りても、DDで自己資本が req_cap を割れば破産する。**証拠金基準で張らない**。
- **lot は常に「現在の自己資本 ÷ req_cap基準」でサイズする**。
- **複利＝実現益で自己資本が増えるたびに lot を再計算して漸増**（幾何成長）。逆にDDで
  自己資本が減れば lot も自動的に縮む（デレバレッジ）＝破産確率を下げる安全装置。
- **過大レバを断つ上限式**（§2）を必ず守る。req_cap基準を超える lot は張らない。

## 0b. ★実口座＝国内 OANDA証券（2026-06-21 確定）— レバ25倍の制約

go-live ブローカーは**国内 OANDA証券**（JFSA登録・MT5・既配線）。税制が圧倒的に有利
（申告分離 一律20.315%＋損失3年繰越 vs 海外の総合課税累進15〜55%・繰越不可）。
ただし**個人レバ上限25倍**が Grid 満玉で律速になる＝§0/§1表の req_cap sizing が
**そのままは使えない**。

- **§1〜§2 の lot 表は高レバ（海外）前提**。25倍では満玉ラダーの想定元本が証拠金を
  超え、**BTが想定しない強制ロスカット**（回復可能な平均回帰レッグを切る＝最悪シナリオ）
  に当たる。よって**実口座 lot は req_cap でなくレバ（証拠金）に合わせて縮小**する。
- **実装（grid_monitor.py）**: `is_live_broker()` 判定で
  ①`LIVE_LOT_PER_PAIR`（25倍安全＝req_cap版S0の約44%）を使用、
  ②`place_order` に**証拠金ガード**（新レベル投入後の証拠金維持率 < `LIVE_MARGIN_LEVEL_MIN`=150%
  なら見送り `live_margin_skip`）、③float_stop/DD閾値は live lot 比でスケール。
- **S0 実口座 lot（自己資本50万・25倍安全）**: AUDCAD 0.15 / CADCHF 0.05 / AUDNZD 0.08 / EURGBP 0.05。
  全4本満玉一方向の必要証拠金 ≈ ¥394k（自己資本の約79%）＝満玉同時でも維持率>100%、
  150%ガードは深部レベルのみ稀に間引く。
- **含意（正直開示）**: 国内25倍では lot が高レバ版の約44%＝**同じ自己資本での想定月利も約44%**
  （50万で中央 約2.4万/月）。さらに**自己資本~75-100万で満玉グリッドが25倍に当たり頭打ち**。
  **月利30万（2.8M満玉）は国内25倍では不可**。本フェーズは「税優位のまま安価に forward-test
  証拠を貯める検証フェーズ」と位置づけ、スケール局面で海外移行を別途判断する。
- **複利・昇格（§2c/§3/§4）の枠組みは不変**だが、実口座 lot は常に「req_cap 安全 ∧ 証拠金
  安全」の**両方**を満たす小さい方で設定（25倍では証拠金側が律速）。
- **接続方式（2026-06-24 診断確定）**: OANDA live は **path_only が -6 Authorization failed** で不可、
  **認証情報渡し（initialize に login/password/server）で接続OK・`trade_allowed=True`**。
  `broker_config.py` の `oanda_live` は path_only を外し credentials 方式（`.env` の `OANDA_LIVE_*`）。
  **symbol_suffix は空 `''`**（OANDA live は AUDCAD 等そのまま）。**4ペア全て取扱いあり**・口座 REAL・
  残高50万・レバ1:25 を `check_broker_connection.py --diag-live` で確認済。

## 2. lot サイジング式（複利で毎月再計算）

```
scale          = 自己資本(円) ÷ 742,000          # 742k = AUDCAD lot=1単位でのバスケットreq_cap_99
AUDCAD lot     = scale × 1.000
CADCHF lot     = scale × 0.305
AUDNZD lot     = scale × 0.552
EURGBP lot     = scale × 0.303
```

- この式が成り立つとき、**バスケット全体の req_cap_99 が自己資本と一致**（破産<1%）。
- **過大レバ上限**: いかなる時も `scale ≤ 自己資本 ÷ 742k`。これを超える lot は張らない
  （証拠金が余っていても）。
- **risk_frac で安全バッファ/加速を調整**: `scale = risk_frac × 自己資本 ÷ 742k`。
  risk_frac<1 は安全バッファ、>1 はレバ加速。**運用は §2c の可変スケジュールに従いペア単位で設定**。
  ペア単位の場合: `そのペアの lot = risk_frac_pair × 自己資本 ÷ 742k × 等req_cap相対比`。

### 2b. risk_frac 感度 — 安全バッファ(<1) と レバ加速(>1) のフロンティア

`risk_frac>1.0` は equity<req_cap で張る＝**レバを効かせて加速**する領域。**速さは破産/半減確率で
買う**取引。数値根拠: `grid_growth_sim_riskfrac.csv`（4本バスケット・複利MC・暦月基盤）。

| risk_frac | eq/req_cap | 月利30万 必要資本 | 到達月(中央) | **P(半減)** | P(破産) | **単月最悪/eq** | 位置づけ |
|---:|---:|---:|---:|---:|---:|---:|---|
| 0.33 | 3.0× | 8.48M | 58mo | 0.0% | 0.0% | 15% | 最保守 |
| **0.50** | 2.0× | 5.60M | 48mo | 0.0% | 0.0% | 23% | **慎重開始(初期)** |
| **1.00** | 1.0× | 2.80M | 21mo | 1.3% | 0.0% | 46% | **plan既定・昇格後の基準** |
| 1.25 | 0.80× | 2.24M | 15mo | 4.8% | 0.0% | 58% | 加速(急ぐ局面の上限) |
| **1.50** | 0.67× | 1.87M | **12mo** | 7.8% | 0.0% | 70% | **レバ活用の実用上限** |
| 1.75 | 0.57× | 1.60M | 10mo | 12.3% | 0.0% | 81% | 非推奨(余白なし) |

- **「単月最悪/eq」が決め手**: 歴史上の単月最悪（1単位 −34.4万）が現自己資本の何%か。**100%超で
  単月だけで口座が飛ぶ**。risk_frac=2.0 で93%・2.5で116%＝ここに**絶壁**があり P(破産)が 0%→37% に跳ねる。
  しかも engine の最悪月はギャップ/FSスリッページ（設定の1.0〜1.3倍出る実績）を完全には含まないため、
  **実機では 1.75 ですら「想定外に悪い1ヶ月で即死」の余白がほぼない → レバの実用上限は 1.5**。
- **risk_frac=1.0** は「破産<1%」のプロジェクト定義そのもの（equity=req_cap_99）。
- 60mo後の自己資本中央値（85M〜1,376M）は「11年エッジが正確に持続した場合」の幾何成長の上振れで
  **bankable でない**。判断は **P(半減) と単月最悪/eq** を軸に。

### 2c. ★「レバを稼いで上げる」可変 risk_frac スケジュール（推奨運用）

不確実性が最大の今（forward-test 未完・全構成 BT 由来）は低 risk_frac で始め、**各ペアが live で
エッジを実証した分だけ risk_frac を解禁**する。安全を前に厚く・加速を後ろに置く。

| 段階 | 条件（ペア単位で判定） | そのペアの risk_frac | 根拠 |
|---|---|---:|---|
| **S0 検証** | live 投入〜forward-test 未合格 | **0.50**（eq=2×req_cap） | BT由来・FS未発火＝生存者バイアス警戒。単月最悪/eq 23%で余白厚い。 |
| **S1 実証済** | §3 forward-test 合格（3ヶ月∧TP≥30∧**FS発火**∧PF>1.2） | **1.00**（plan既定） | エッジが live・損切り発火後も PF>1.2 を確認＝req_cap基準を許可。 |
| **S2 加速(任意)** | S1 を**さらに3ヶ月**維持＋実DD≤MC中央 ＋ 半減を金銭的/心理的に飲める | **1.25〜1.50** | 到達21→12-15mo。代償 P(半減)5-8%。**1.5を超えない**。 |

- **判定はペア単位**: AUDCAD が S1 でも CADCHF が S0 なら、その月の各ペア lot は
  `lot = (riskfrac_pair) × 自己資本 ÷ 742k × 等req_cap相対比`（ペアごとに risk_frac を差し替え）。
- **降格もする**: あるペアが §7 撤退条件に触れたら risk_frac を一段下げる（S2→S1→S0）か停止。
  「上げたら戻さない」はしない。**実証が崩れたらレバも返上**。
- **全体の上限ガード**: どのペアも **risk_frac ≤ 1.5** を絶対上限とする（§2b の絶壁回避）。
- **入金との関係**: 入金は risk_frac を上げずに scale を上げられる**最も安全な加速**（分子=lotと
  分母=耐久資本を同時に増やす）。**急ぐなら、まず入金 → それでも足りなければ S2 のレバ**、の順。

### 2d. ★実口座（国内OANDA 25倍）の S0→S1 ロット移行ルール（2026-06-24 確定）

§2c の risk_frac 表は**高レバ前提**。国内OANDA25倍では証拠金が律速のため、実口座 lot は
**§0b の証拠金安全 lot（`vps/grid_monitor.py LIVE_LOT_PER_PAIR`）を基準**に、下記ルールで移行する
（§2c を上書き）。数値根拠は `grid_live_margin_concurrency.py`（11年 同時オープン証拠金分布）と
`grid_live_lv3_review.py`（浅化は不可）、証拠金モデルは OANDA証券公式（下記）。

**確定した証拠金モデル（重要な前提）**:
- OANDA証券 MT5 の必要証拠金 = 取引数量 × **前日終値** × 4%(FX個人=25倍)。**日中は固定**、メンテ
  時間（5:55–6:05 / 標準6:55–7:05 JST）に**1日1回だけ前日終値で再計算**。日中の価格変動では既存
  建玉の必要証拠金は**変動しない**（出所: help.oanda.jp/oanda/info_and_news/show/154, oanda.jp/info/1395）。
- ゆえに**日中の同時ラダー構築（concurrency tail の本体）は実行時ガード（維持率<150%で新レベル
  見送り `live_margin_skip`）が完全制御**＝日中ストップアウトは構造的に防げる。**base lot は破滅
  ダイヤルでなく「通常時PnL＋tail間引き頻度」ダイヤル**。破滅回避はガードが担う。

**11年の同時オープン証拠金 実測（3本・自己資本50万・25倍概算）**:
- 全3本同時に片側満玉（深度≥4）は **11年で0回**＝「全ペア満玉」は非現実的。
- だが史実ピーク合算証拠金 = **421k = 維持率119%**（EURGBP両建て＋組合せで片側満玉357kより高い）。
  中央値58k(868%) / p95 188k(266%) / p99 266k(188%) / p99.9 371k / max 421k。
- ＝現行 S0 lot は「満玉前提で過剰に絞った」のでなく、**史実tailにちょうど耐える水準**。

| 段階 | 条件（ペア単位） | base lot(自己資本50万) | 証拠金ガード | 根拠 |
|---|---|---|---|---|
| **S0 検証** | live投入〜§3未合格 | AUDCAD0.15 / CADCHF0.05 / EURGBP0.05 | **150%** | forward-test の BT忠実性優先。史実ピーク維持率119%に耐える |
| **S1 実証済** | §3 合格（3ヶ月∧TP≥30∧**FS発火**∧実現PF>1.2）| **×1.25 = 0.19 / 0.06 / 0.06** | **150%（据置）** | p99同時証拠金がガード水準＝上位~1%の稀局面のみ間引き。通常時PnL増。DDは req_cap の55%で依然安全 |

- **スケール**: 入金時は上表を `自己資本 ÷ 50万` 倍（比率維持で維持率一定・§1入金早見表）。S1の×1.25は
  50万基準の相対倍率。判定・移行は**ペア単位**（そのペアが§3合格で ×1.25、他は S0 据置）。
- **⚠️証拠金ガードは150%を下げない（100%は厳禁）**: ガード閾値＝**翌営業日メンテ時の日次再計算に
  対する唯一のバッファ**（日中固定ゆえガードは日中しか効かず、翌朝の再計算による既存建玉の証拠金
  ステップ変化だけは介入不能）。150%は翌日再計算で維持率100%割れに**約50%の逆行を要す**（ブラック
  スワン級）。**100%にすると翌朝のわずかな逆行でメンテ時ロスカット**。「維持率100%まで許容」の意図は
  **base lot 増（×1.25）で消化**するのが正解＝ガードを下げても間引かれるのは最稀~1%かつ**深部の回復
  レッグ（平均回帰の稼ぎ頭）**なので、エッジ増は僅少・失う翌日バッファは甚大＝悪いトレード。
- **さらに攻める場合（任意・上限）**: base lot は **×1.5（0.22/0.075/0.075）まで**許容。ただしガードが
  tail 上位数%で深部レッグを間引き**実現PFが BT から乖離**するため、実測PFで監視。ガードは常に**≥150%**。
  lv3浅化での証拠金捻出は**不可**（`grid_live_lv3_review.py`: AUDCAD wfoMin1.20→0.91・req_cap悪化）。
- **降格**: §7 撤退条件に触れたら base lot を S0 へ戻す（ガードは常時150%据置）。

## 3. ペア昇格 / lot 増プロトコル（forward-test 合格条件 — 全て満たす）

1. 稼働 **3ヶ月以上**。
2. **TP決済 ≥ 30件**（薄標本回避）。
3. **float-stop / B48 が最低1回発火**（損切り発火前の黒字は生存者バイアス。GBPJPY demo
   は28決済 WR100% でも FS発火0回＝信用しない、CHFJPY教訓）。
4. 発火後も **実現PF > 1.2**。
5. FSスリッページ ≤ 設定の1.3倍（ギャップ貫通が想定内）。

- 合格したペアのみ lot を req_cap 基準まで引き上げる。未合格ペアは**検証 lot（0.01）据置**。
- **★発火頻度が低いペアは達成に時間がかかる前提**: AUDCAD は活動月 49%（アイドル 6.1ヶ月/年）
  ＝ TP≥30件 や FS発火に**数ヶ月〜年単位**かかりうる。だから demo は4本並行で回し、合格した
  ものから順に live lot を引き上げる（全員の合格を待たない）。
- Tier順序: **Tier1 AUDCAD最優先**（capEff 1.40）→ Tier2（CADCHF 0.43 / AUDNZD 0.18 / EURGBP 0.14）。

## 4. 複利・再投資ルール

- **再計算タイミング**: 月初に前月の**実現損益**を自己資本に反映し、§2の式で全ペア lot を再計算。
  含み益は反映しない（実現のみ＝生存者バイアス回避）。
- **組み入れ単位**: lot は broker 最小（0.01）刻みで切り捨て。端数は次回反映。
- **安全バッファの引き出し**: 自己資本が 1.5M を超えたら、超過分の一部（例：四半期ごとに
  実現益の20%）を口座外へ退避するか保持するかを選択。退避すれば破産耐性が上がるが複利は鈍る。
  **推奨：2.8M 到達までは全額再投資（複利優先）、到達後は月30万を生活費として引き出し残りを複利**。

## 6. 決定論的 killswitch（統計的予測は全期間で不成立と確定済）

統計的killswitch（実DD vs MCエンベロープ）は5期間×2統計で検知力0-18%・遅延540-1900日＝
**全期間で不成立**（FSファットテールがエンベロープを巨大化させドリフト枯渇を数年隠す）。
よって**予報に頼らず決定論で監視**する：

1. **執行整合性チェック**: 注文が設計（atr/lv/ci/fs/regime）通りか毎日確認。乖離＝即停止・調査。
2. **FS単発スリッページ > 設定×1.3** → 当該ペア停止（ギャップ貫通が想定外）。
3. **自己資本が req_cap_99 を割ったら全停止**（ハードストップ）。
4. **構造（10年IS）が壊れた時のみ年単位で降格**。緩慢なエッジ劣化は数年識別不能と受容。
5. **★活動性チェック**: 4本とも一定期間（例 **60日**）完全ゼロなら、ゲート設計通りでなく
   **CI計算 / データ供給の別問題**を疑い点検（単独ペアの数週間ゼロは正常＝二重ブロック）。

## 8. 正直なリスク開示

- **50万は月利30万に必要な 2.8M の約18%**。30万/月は**複利での到達点**であり、初期数ヶ月の
  月利は**数万円規模**（50万なら中央約5.4万/月）。**到達中央 21ヶ月**（risk_frac=1.0）。
- **高CI窓が来ない月はゼロもありうる**（AUDCAD 3週間ゼロ実例）。バスケットでもアイドルは
  年0.4ヶ月だが、個別ペアは年5〜6ヶ月アイドル。
- **最悪シナリオ**: 5年後 P5（下振れ5%）でも自己資本 **約1,107万**（バスケット, risk_frac=1.0）＝
  元本割れ確率 0.0%・破産 0.0%。ただし P1 近傍では**自己資本半減（250万割れ）を 1.3% で経験**しうる。
- **数値の前提**: 11年（2015-2026 Dukascopy）の分布が将来も続くこと・執行スリッページが
  engine 想定内であること・エッジが構造的に劣化しないこと（劣化は数年識別不能）。
  median 8,531万（5yr）等の上振れは**「11年エッジが正確に持続した場合」**であり楽観側。
  **判断は下振れ（P5）と決定論 killswitch を軸に**。
